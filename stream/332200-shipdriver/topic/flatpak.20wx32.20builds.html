Whatever you do with the branches, the tag is still there, it does not depend or belong to any branch. Nothing bad in that, but so you know.

Try to rebase and merge. Checkout your temporary branch, then do git rebase master. This time, if there are conflicts you should be able to sort them out, you have written at least one part of any conflict. But most likely, it will pass without conflicts.

After rebasing, do git checkout master; git merge <temporary branch> . This is the way to handle a "feature branch" which is exactly what you have here. After merging the branch, remove it on github to keep it tidy.

BTW: Have I talked about using "Rebase and merge" when you merge PRs? It's an option on the dropdown in the big green button "Merge", far superior to a plain merge.
