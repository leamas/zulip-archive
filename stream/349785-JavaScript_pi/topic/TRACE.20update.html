For information only, I have rewritten the TRACE feature from next build JS v2.0 alpha 4.
(1) whether or not tracing is compiled in. now depends on
#define TRACE_YES true // if true, include TRACE code. Should be false in release
If false, only ((void)0) is compiled (needed because of the following ; )
This has silenced the numerous warnings from the compiler.
(2) TRACE(0, ... is always traced, otherwise tracing is now between two limits

#define TRACE_MAX   30  // TRACE level range   (Level 0 always traced)
#define TRACE_MIN   25

which helps make tracing specific to the area of interest
