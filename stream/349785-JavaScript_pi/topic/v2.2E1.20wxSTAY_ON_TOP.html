Tony Voss (Apr 12 2023 at 13:05):

I have now converted the JavaScript plugin to use wxFrame instead of wxDialog. It can now use wxFLOAT_ON_PARENT. Given different views on whether keeping consoles on top is dsirable, I have added a user option to choose whether or not to use wxFLOAT_ON_PARENT.
The change-over was much easier than I had anticipated.

  1. In wxFormBuilder add a new wxFrame and move it to the top of the window list.
  2. Drag the top sizer from wxDialog to the new wxFrame.
  3. Delete the now empty wxDialog and rename the wxFrame.
  4. Adjust the type of any wxDialog event processing to be wxFrame processing
