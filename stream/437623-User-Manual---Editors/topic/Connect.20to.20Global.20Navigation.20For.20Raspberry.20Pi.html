It is exactly the same thing. The Linux screenshot is actually from an Orange pi 5, an SBC very similar to Tpi 5.

This is a general observation. There is no "Rpi" use case in any scenario besides boot/install. The actual use case is Debian for most users on Rpi i. e., those using Raspbian.

But in the end, the Rpi supports all kinds of distros like vanilla Debian, Arch, Ubuntu, Fedora and what not. What we support is the same as for PC platforms:

  1. Native Debian,deb packages to be used on Debian and derivatives like Raspbian and LTS Ubuntu release.

  2. Flatpak to be used on other Linux distros.

Note that there basically are no such things as "Linux' or "Rpi" here. It's either Debian or Flatpak. . Full stop.
