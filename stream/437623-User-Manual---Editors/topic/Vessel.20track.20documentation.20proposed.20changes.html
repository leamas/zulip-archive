Mike Guryan (May 20 2024 at 21:33):

@Terry Slattery -

First question, why Automatic Daily Tracks exists, I.e., why reset daily. I can think of several reasons:

  1. If OCPN was left open and tracking left on for an extended period, the size of a GPX file for a single track could concievably exceed usable/useful limits.
  2. Ship's logs typically cover a daily period, whether coastal cruising or on a passage. Locating position information in a track is simplified if you know which date it occured on. I can't imagine trying to find where I was on a long passage if it was in a single GPX file.
  3. Rotating the track log file daily was probably far simpler than trying to rotate it on a per-watch basis.

I would add the following to Route from Track:

Creates a route from track data. The new route will appear in the Route & Mark Manager and will have the same name as the track. Depending on the precision used to create the track, it is recommended to use the Reduce Data function on a track prior to converting it to a Route. This will reduce the number of waypoints.

The Use Tracks reference to Track Precision should be a link to Own Ship#Tracks?
A link would be good. I need to better understand how to create bookmarks in the editor so that more fine-grained links can be created. I would have created one if I knew how.

In general, the purpose of the Tracks Tab page in the manual is to provide a basic description of the controls. The Use tracks page is to provide more detailed information and use cases.

The fact the the controls for Automatic Daily Tracks is under Ships>Own Ship pretty much requires that we at least describe the options there, with more details and use cases provided in the Use Tracks page. In a similar vein, the fact that the Tracking toggle is i't own button on the toolbar requires that we describe it there.

I can see consolidating the Tracks Tab and Track Properties pages, but it will be long. That's my primary reason for separating them in the first place.
