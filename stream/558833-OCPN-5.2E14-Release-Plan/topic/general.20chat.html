Håkan Svensson (Mar 23 2026 at 14:26):

@Dave Register @ All
Refereeing to this CF post about changes for 5.14.
From gitk I selected some items and made a list. We can use as is, or revised, on the download page and CF?

A selection of improvements and measures implemented for OCPN v.5.14.

  • Allow NMEA0183 sentences without Checksum.
    If present it must though be correct.

  • Introduce and implement Plugin API 1.21 for multi functional plugin handling.

  • Correct fault in GPX file import with file name containing UTF-8 chars, Android.
  • DataMonitor: Filter selection bugfix and several feature updates.

  • A fairly extensive revision of instructions for multiplexes and communications. 
    However, nothing that is directly visual to a user.

  • Improve ENC sounding rendering performance.

  • Correct auto-follow behavior and other handling on touch screen interface.
  • Enable MMSI Properties dialog in options->ships for Android.
  • Refine the selection method for quilted charts.
  • Handle invalid time cases in new route.
  • Add N0183 sentence $xxTHS for true heading.
  • Fix RangeRingColours for Isolated Marks.

  • Improve tide station window.
    Tides are updated for reliable real time presentation.
    Enable selection of tide/current graph by double-click.
    Add tidal currents for the Baltic and North Sea.

  • Correct AIS ATON icon rendering logic.

  • Add a search ship name function to the AIS target list.
  • Refactor tooltips for the chart canvas.
  • Fix input of negative latitudes / longitudes.
  • Restrict Notification button "Bell" to primary canvas.

  • Implement delayed AIS alert dialog and sound effect to reduce disruptive false alerts. 
    Improve AIS Alert deferral logic on repeated alerts

  • Refactor/simplify auto daily track restart logic

  • Streamline the rebuilding of the chart database  
    Improve deferred chart update progress dialog

  • Remove deprecated option for COG/SOG estimation from position fixes

  • Correct AIS Query handling of message 14 Safety Broadcast Message

Dave Register (Mar 27 2026 at 02:32):

News...
Finally worked out the trouble with circular routes and last leg display. #5144
Turns out that we made a change to the navobj_db structure very soon after the release of O5.12.4. We knew at the time that this change would require an automatic migration update of existing O5.12.4 database when O5.14 was released.
And to be honest, in the fog of time, we just forgot about that required migration update step. It slipped off all lists, and simply was not implemented in O5.14.x testing.
We have done the migration framework and implementation in OCPN master branch now, and it is ready for merging to v5.14.x branch.
As much as I regret it, we will need to do a Beta-4 test, since this migration step will theoretically happen with virtually every user of O5.12.4. So a new Beta-4 will be coming soon.
Thanks for your understanding.
