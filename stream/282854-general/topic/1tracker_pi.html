Hi.

After 9 years, I finally succeeded and changed my experiment into a working plugin. I named it 1tracker. This plugin allows users to send positions periodically to tracking websites. For example every 10 minutes to noforeignland.com, every minute to a personal website, and so on. Upon request other default trackers can be added to the codebase as well.

The plugin is mainly meant for cruisers that run OpenCPN and have permanent connectivity (ie via Starlink). To also support the group that only occasionally has internet, I strive to add some extra functionality over the next couple of days.

Currently beta versions are ready for testing. Some other OCN users (in the Dutch sailing community) said they will do so. I tested myself for Mac and Windows. One thing I'd like to share here, since I ran into an issue for the Windows version.

That had to do with std::mutex. I managed to solve it, but that took quite some time and debugging. Other plugin developers might run into that as well, when developing new stuff or upgrading the codebases of the existing plugins.

Now, please feel free to comment on the chosen solution. Perhaps things could have solved easier or better? More than willing to learn.

The plugin itself is only on my personal github (pa2wlt). Rick had some issues with adding me to the community, be he will succeed with that, and I will do a pull request and do what is necessary to get the plugin in the catalog.

Regarding the std::mutex please see here: https://github.com/pa2wlt/1tracker_pi/blob/master/docs/vs2022-17.10-mutex-writeup.md

Best, from Biskaje.
