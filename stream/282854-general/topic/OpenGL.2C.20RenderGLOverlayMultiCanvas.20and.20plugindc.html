There is the bundled library plugindc that simplifies drawing by providing a simple abstraction layer. It allows the developer to use its methods to draw using either OpenGL (wxGLCanvas) or non-OpenGL (wxDC). For OpenGL the abstraction means a developer can write:

Drawrectangle(x, y, width, height);

insead of low level complex wxGLCanvas API's

glBegin(GL_QUADS);
glVertex2i(x, y);
glVertex2i(x + width, y);
glVertex2i(x + width, y + height);
glVertex2i(x, y + height);
glEnd();

However most plugins seem to implement a class that inherits from plugindc that provides a couple of members and methods. Eg from DashboardSK

class dskDC : public piDC {
private:
    /// Scale factor of the context
    double m_scale_factor = 1.0;
    bool m_is_gl;

with constructors for either wxGLCanvas or wxDC and getters/setters for scale factor etc.

What is the reason or the requirement for this inherited class ?
