Alec Leamas (Apr 07 2022 at 14:00):

If you are really sure that you a) have the correct contents in your local master branch and b) that noone is using your repo besides you you can force-push using git push -f origin master:master. This will effectively overwrite the history in your remote repo, and break the overall git promise that nothing ever is lost. But it will work

If you have the correct content in the remote branch you can do like this:

$ git checkout -b tmp
$ git remote update origin
$ git branch -m master master.old
$ git fetch origin master:master
$ git checkout master

This will give a master branch which is a copy of the remote master branch, saving you local master branch as master.old. Here, everything is still available, but you need to be careful when re-constructing master.

Don't copy files. It's generally a bad idea, and even more so on windows where you more or less always get permissions problems
