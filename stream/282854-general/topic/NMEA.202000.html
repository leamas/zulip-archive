Dave Register (Mar 08 2023 at 01:36):

Kees...
With this API, a plugin should be able to send arbitrary PGNs to the N2K bus. Theoretically, then, we can build a plugin that emulates a commercial N2K autopilot. Or there may be other ways to interface with N2K autopilots.
I think there is/was some discussion of a Raymarine Autopilot interface of some sort, but I lost the thread on that. IIRC, a plugin would synthesize the proper PGNs to allow the autopilot to follow a route created on OCPN. And the PGNs are proprietary, so some RE required.
Am I on the right track? If so, we can start a separate discussion about this.
The key point is this: We do not necessarily need a Raymarine A/P plugin now. But we do need to validate/prove that the API in O571 is functional and complete enough to build such a plugin, before we release O58x. So some "proof of concept" work would be very useful assurance of that capability.
