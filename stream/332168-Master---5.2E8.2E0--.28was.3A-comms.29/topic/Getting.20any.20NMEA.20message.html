Topic: Getting any NMEA message

Tony Voss (Apr 27 2023 at 12:18):

The new messaging architecture allows for the creation of a message-specify method to handle a given message, e.g. through the NMEA0183Id and NMEA2000id.
This makes for clean code if you know what messages you want to handle.

But it does not (I think) make it possible to examine all traffic, when you might want to look at what is being sent on an NMEA network.
For example, there is a JavaScript MessageNMEAsampling that examines all NMEA0183 received over the old mechanism and catalogues it.

Is this possible with the new architecture? Or does it require, perhaps, a special NMEA0183Id to receive anything?

NMEA2000
I have heard of users wanting to receive all manner of NMEA2000 messages (engine revs, fuel gauges etc. etc.). This has, rightly, been turned own as irrelevant for a navigation program. It would lead to severe bloat.
But it could be possible for users to meet such requirements via JavaScript in which they could do their own thing.
I hope to find time to look at decoding non-OCPN-supported PGNs.

But like with NMEA0183, it would be useful to be able to receive and catalogue all NMEA2000 traffic.
Is it possible to receive any NMEA2000 message? Or does it require a special NMEA2000id for this purpose?

SignalK
Might the same issue arise for SignalK? I am not familiar enough with it to know.

Do the above requirements need additions to the present capabilities? I could raise RFEs in the light of input here.
